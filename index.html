General Summary

History of Python

Python was made by Guido can Rossum in 1989 at CWI! No, not that CWI. Cebtrum Wiskunde and Informatica, in Amsterdam. Guido built the language, inspired by languages like Java, Lisp, Perl, Haskell and C, to try to create something just as powerful and useable, while also making it as easy-to-read and easy-to-write as possible. Bringing powerful languages down to a level where a less skilled programmer can still wield all the power a language like C++ or Java can have, without all the annoying intricacies or weird formatting to hinder them.

The language is known for being a bit of a one-size-fits-all language, useable in a lot of different programming situations. Scripting, programming, machine learning, graphics processing, and all kinds of other things. However, instead of making the language with all this functionality out-of-the-box, Guido decided to instead focus on making Python an incredibly extensible language. Combined with it's focus on ease of use, as well as readability, it makes Python one of the most popular and most versatile language in the world.

According to 'Zen of Python', written by Tim Peters, Python has many key guiding principles:

  • Beautiful is better than ugly.
  • Explicit is better than implicit.
  • Simple is better than complex.
  • Complex is better than complicated.
  • Flat is better than nested.
  • Sparse is better than dense.
  • Readability counts.
  • Special cases aren't special enough to break the rules.
  • Although practicality beats purity.
  • Errors should never pass silently.
  • Unless explicitly silenced.
  • In the face of ambiguity, refuse the temptation to guess.
  • There should be one—and preferably only one—obvious way to do it.
  • Although that way may not be obvious at first unless you're Dutch.
  • Now is better than never.
  • Although never is often better than right now.
  • If the implementation is hard to explain, it's a bad idea.
  • If the implementation is easy to explain, it may be a good idea.
  • Namespaces are one honking great idea—let's do more of those!
  • Use proper indentation or I'll break your kneecaps.

(...Ok, maybe not that last one.)

So, while it's not the most technically powerful language in the world, it's extensibility combined with it's user-friendly nature makes it one of the most effective languages in the world.

Where Python is used today

Have you ever used Wikipedia? How about Google? Facebook? Spotify? Reddit? Well, congratulations, all of those sites use python in their websites. In fact, Reddit is implemented in Python entirely! It's versatility makes it a natural choice for most websites.

Python is included in standard operating system MacOS, as well as almost every distribution of Linux.

Python is even used by NASA! Meeting the United Space Alliance's requirements of a language being 'fast, cheap and light', Python is used by NASA today to do secret-NASA things. They don't tell anyone exactly what they're doing with it, but they're NASA, so that makes sense.

The Raspberry Pi (a computer about the size of a wallet, costing less than $40) has the word 'Pi' in it's name not only as a clever play on words, but as a reference to Python! Python is a primary language used on the Pi, and a majority of the systems on it are built and optimized for Python programs and programming.

Python has even gone on to inspire many other languages, such as Groovy, a language built to make Java more Python-like. Other inspired languages include Cobra, CoffeeScript, Go, Swift, and many others. Specifically, those languages usually follow in Python's footsteps in terms of syntax, indentation style, and ease-of-use.

Python's programming paradigms

Python supports four main programming paradigms: imperative, functional, procedural, and object oriented. This page will only be highlighting the object oriented and procedural paradigms.

Object oriented Python

Python is used as an object-oriented language (OOL). Object-oriented programming is a unique and difficult concept, but to put it simply: it acts on the idea that everything in a given program is an 'object', with properties and functions that can be the same as other objects, but act independently. It saves on resource usage, while also making the program much easier to manage. In an OOL, if you want there to be two different cars in your program, you just call two Cars(), and differentiate them. In a non-OOL, however, you have to create the codebase and the programming for each car independently, which takes more time, disk space, and resource usage when running the program. This is a simple explanation, of course.

Procedural Python

Python is commonly used as a procedural programming language in scripting environments. This paradigm is a subset of imperative programming in which statements are structured into procedures. This paradigm is typically a list of instructions that execute one after the other starting from the top of the first line of the program. This is useful for creating modules that can be unrelated to other modules and programs yet still be accessed by said programs and work together with other modules to achieve goals for the program. As well as support reuse of procedures within a program that are defined in either other libraries or modules.

Quirks and advantages

No language is perfect, but every language has it's specialties, and Python is no exception. What some would consider downsides, other consider features. Here's a few examples of how Python is a bit different than other, similar languages:

  • It's extreme Popularity makes it a bit of a token choice at times, but also makes it compatible with countless codebases and modules people have made in the past
  • It's fundamentally Object-Oriented nature makes it useable in modern object-oriented programming styles, and especially powerful when dealing with large amounts of objects or data.
  • Python features Automatic Garbage-Collection, making it easy for people to toss large amounts of data around without having to worry about data leaks.
  • It's Ease-of-Use as well as it's relatively Forgiving Learning Curve makes it ideal for beginning programmers, or those learning the language for a project.
  • It's Expansive Standard Library makes it incredibly powerful even without any expansions, making it incredibly useful out-of-the-box.
  • Python is an Indentation/Whitespace Sensitive language, meaning the indentation style is important and fundamental to how it's written. Many people don't like this, but it removes the need for many formatting items such as brackets ((){}[]) in certain scenarios.

Syntax (needs expanded on but we need to consider what is to be documented here or in our examples)

Python's written syntax is relatively simple, with one key thing that many languages do not have: significant indentation!

While good indentation in other programs is recommended for readability, in Python it's also important for the compiler to read it too. Where you put your tabs and why is very important. For example, everything contained in a 'while' loop or an 'if' statement must be indented within it, or else it assumes it not to be part of that loop.

x = 2
if x==2:
print("I'm in the if!")
print("I'm not in the if!")

Or:

while x < 5:
print("I will print 5 times!")
x = x + 1
print("I will print once!")

Aside from that, Python syntax is pretty straight-forward. Creating a variable is done with [name] = [value], such as:

HowOldAmI = 22

And Python is built to understand what variable type it should be! In this instance, HowOldAmI would be an integer, with the data '22' stored inside it.

Comments are easy too! Just use the # character to leave a comment, and the computer will ignore whatever you throw in there.

while x < 5: #I am the ghost of development past!

Control flow statements

When does the code learn what code to run next?

IF

Good ol' IF.

Syntax:

x = 2
if x==2:
print("I'm in the if!")

IF is used to check a conditional statement to know whether or not to run a set of code. It checks to see if a particular thing is true or not, and if it is, it runs whatever is indented in front of it.

May contain an optional 'else' statement, which is a set of code to run instead if the conditional isn't true.

FOR

FOR is a bit more complex. FOR [things] IN [list]: [do stuff to things]

It runs over the same code over and over again for a list of items in an array.

Syntax:

for x in range(1,8):
print(x, "bottles of beer on the wall")

May contain an optional 'else' statement, which runs when the FOR loop finishes.

WHILE

The WHILE is like an IF if it got stuck, like a record player. It keeps doing that little bit of code, over and over again, as long as it's conditional is still true.

Syntax:

x = 2
while x < 8:
print("I'm in the while!")
x = x + 1

May contain an optional 'else' statement, which runs when the WHILE loop finishes.

Functions

A function is a method, essensially. When a certain keyword() is called, it does some code. To define a function, use 'def'

def do_the_thing():
print("I'm doing the thing!")
do_the_thing()

And to call, simply use do_the_thing().

Arguments can be added as well, which are variables you can pass into a function to make it do stuff! simply put something in the brackets (()) when you define it, and something in the brackets when you call it, and it will pass that data from the call to the variable you define in the def function.

def do_the_thing(thing):
print("I'm doing the", thing, "!")
def do_the_thing_2(too, thing):
print("I'm doing the", thing, too, "!")
do_the_thing_2("too", "thing")

You can even have a function take an unknown number of variables by using the * symbol like this: do_something_else(*stuff)

Functions can be called within other functions, with different data, to make big, complex programs! Also, there's many other ways to format the input paramaters and output of a function. You can make it take a dictionary, you can make it have default values, you can use the return keyword to have it return a value when called, and you can use the pass keyword to have a function do... nothing.

Object oriented programming

Object-oriented programming is the idea that code is composed of many different types of classes and objects, which all have inherint and inherited code from it's parent class, to save on data and space. In an object oriented language, if you want there to be two different cars in your program, you just call two Cars(), and differentiate them. In a non-object oriented language, however, you have to create the codebase and the programming for each car independantly, which takes more time, disk space, and resource useage when running the program.

A 'class' is a blueprint with which to create objects. For instance, if I wanna make the 'car' class, I simply do this:

class Car:
pass

And if I want things to be added to that car when I create a car, I define the __init__ function:

class Car:
def __init__(self, color, model):
self.color = color
self.model = model

With that class, I can create a new car object by saying x = Car("blue", "Mitsibishi")

We can define functions in that class to act on the paramaters inside it. For example:

class Car:
def __init__(self, color, model):
self.color = color
self.model = model
def printColor():
print(color)

...Then we can call it with:

x = Car("blue", "Mitsibishi")
x.printColor();

And we can even make smaller child classes from existing classes! Simply put the parent class in the parentheses and voila! Child class. Example:

class Sportscar(Car):
pass

...And that child class can have it's own paramaters and functions.

Procedural programming

As stated earlier python supports the paradigm of Procedural programming which relies on procedure calls to modularize code and help simplify a program by by breaking code into smaller pieces that can later be called and used collectevly by a developer. Procedural programming is an older form of application development but is still very useful for solutions that lend themselves to step-by-step execution.

Python supports this very much so in scripting environments such as Jupyter Notebook or Cocalc of which written and module procedures are commonly all that is used in a program.

A simple example of this programming paradigm would be as follows:

x = [1,2,3,4,5]
def average(list):
tmp = 0
for i in list:
tmp = i + tmp
return tmp/len(x)
print(average(x))

Variable scope and lifetime

Python has three levels of scope (or four, depending on who you ask), though usually people think of it as two. Global and non-global. However, let's look at all three individually:

  • Local/Enclosing (shortest)
  • Global
  • Built-in (longest)
Local/Enclosing

The shortest and fastest of the four. If a variable is defined in a function, it only exists within the function, and is destroyed (or at least becomes unreachable until it's garbage-collected) once the function ends.

Since it's the shortest and cheapest scope, you wanna try to make as many of your variables you can manage in this scope. Cluttering memory with variables you only need once can be avoided by using local scope.

Similarly, if there is a function within another function, all the variables in that inner function are enclosed within that inner function, and cannot be accessed directly by the surrounding function that it's in. If you have an outer() and an inner() function, inner() can access variables in outer(), but outer() can't access variables in inner()

Global

Global variables can be seen, accessed, and changed anywhere in the program. If you've got a thing that's too important and too integral to the program to pass between function paramaters, make it global. Make it big and loud enough that the rest of the program can just know what it is and where it is.

Global variables are either declared outside of a function, or by using the global keyword when defining it within a function. While this can be tricky, as you don't want to accidentally call on a global variable that hasn't been declared yet, it can be useful in a pinch when you gotta make a new variable that the whole program can see.

Built-in

Essensially, this is the scope that all Python keywords fall under, and everything in there has already been defined for you.

You don't wanna have to go through with every program and define what things like True and while mean, so this scope is where all of those are sitting. If you're messing with this scope, you'd better know what you're doing.

LEGB Rule?

The LEGB is the idea that if you tell Python to find the x variable, it will first look in the Local scope, then the Enclosing scope (in any function that this function may be nested in), then in the Global scope, then finally in the Built-in scope, in that order. If it finds it in one of those scopes, it will use whatever it found first. If it doesn't find it, it will catch your computer on fire. I'm kidding, of course, it will actually just throw an error and kill your program.

In general, you want your variables to be as far up (towards local) in that order as you can, to save on resource space in your program.
